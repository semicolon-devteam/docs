# 주간 리포트 시스템

SEMO 기반 PO/운영자 주간 리포트 자동 생성 시스템

## 리포트 종류

| 리포트 | 대상 | 주요 내용 |
|--------|------|----------|
| **PO 주간 리포트** | Product Owner | Epic/태스크 진행률, 스프린트 목표, 우선순위 현황 |
| **운영자 주간 리포트** | 운영팀 | 버그 현황, 피드백, 배포 이력 |

## 자동 생성 스케줄

- **주기**: 매주 월요일 오전 9시 (KST)
- **방식**: GitHub Actions
- **범위**: 직전 주 월요일 ~ 일요일

## 수동 실행

### GitHub Actions에서 실행

1. Actions 탭 이동
2. "Generate Weekly Reports" 워크플로우 선택
3. "Run workflow" 클릭

### 로컬에서 실행

```bash
cd docs/reports/scripts
npm install
npm run generate
```

### 환경 변수

```bash
export SUPABASE_URL="your-supabase-url"
export SUPABASE_SERVICE_KEY="your-service-key"
export GITHUB_TOKEN="your-github-token"
```

## 커스터마이징

### 프로젝트 추가/변경

`generate-reports.ts`에서 `config.projects` 수정:

```typescript
const config: Config = {
  // ...
  projects: ['랜드', '오피스', '새프로젝트'],
};
```

### 스케줄 변경

`.github/workflows/generate-weekly-reports.yml`에서 cron 수정:

```yaml
schedule:
  - cron: '0 0 * * 1'  # 매주 월요일 00:00 UTC
```

## Slack 알림

리포트 생성 완료 시 Slack 알림 전송 (선택사항)

### 설정 방법

1. GitHub Repository Settings > Secrets 이동
2. `SLACK_WEBHOOK_URL` 시크릿 추가
3. Slack Incoming Webhook URL 입력
