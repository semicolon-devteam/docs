🔌 API 명세

Method Endpoint 설명 인증
GET /api/community/posts 피드 조회 (페이지네이션: ?page=1&limit=20) 선택
POST /api/community/posts 게시글 작성 (body: content, media_urls[]) 필수
GET /api/community/posts/[id] 게시글 상세 + 댓글 조회 선택
POST /api/community/posts/[id]/like 좋아요 토글 필수
POST /api/community/posts/[id]/comments 댓글 작성 (body: content) 필수
DELETE /api/community/posts/[id] 게시글 삭제 (본인만 가능) 필수
GET /api/widgets/trending Trending Topics 조회 선택
GET /api/widgets/dealer Dealer 위젯 데이터 (listings, inquiries) 필수 (Dealer)
GET /api/widgets/buyer Buyer 위젯 데이터 (saved, searches) 필수 (Buyer)
POST /api/upload/image 이미지 업로드 (Supabase Storage) 필수
📌 Note: 모든 API는 Authorization: Bearer <token> 헤더 사용 (인증 필요 시)

⚠️ 예외 처리 테이블

시나리오 UI 동작 메시지
비로그인 사용자가 Post 버튼 클릭 모달 팝업 "Please log in to post"
게시글 500자 초과 Post 버튼 비활성화 + 경고 텍스트 "Maximum 500 characters"
이미지 5개 이상 업로드 경고 토스트 "Maximum 4 images allowed"
네트워크 오류 (게시글 로딩 실패) 재시도 버튼 표시 "Failed to load posts. Retry?"
이미지 업로드 실패 업로드 칸에 오류 아이콘 "Upload failed. Try again?"
본인 게시글 삭제 확인 모달 "Are you sure you want to delete this post?"
타인 게시글 삭제 시도 403 에러 토스트 "You can only delete your own posts"

📱 반응형 & 접근성 가이드

Breakpoints

Viewport Layout 변경사항
1440px+ 3-Column Left Nav (240px) + Content + Widgets (320px)
1280-1440px 2-Column Widgets 숨김 (Fly Menu로 접근)
1024-1280px 2-Column (Collapsed Nav) Left Nav 자동 Collapse (80px)
768-1024px 1-Column + Collapsed Nav Content만 표시, Nav는 아이콘만
< 768px Mobile Layout Bottom Nav + Hamburger Menu

접근성 (WCAG 2.1 AA 준수)

  • 키보드 네비게이션: 모든 인터랙티브 요소 Tab 접근 가능
  • 포커스 표시: 포커스 시 outline 표시 (2px solid #1da1f2)
  • Alt Text: 모든 이미지에 alt 속성 필수
  • ARIA Labels: 아이콘 버튼에 aria-label 추가
  • 색상 대비: 최소 4.5:1 (텍스트), 3:1 (UI 요소)
  • 스크린 리더: 시맨틱 HTML 사용 (nav, main, article, aside)

성능 최적화

  • 무한 스크롤: TanStack Query Infinite (20개씩 로딩)
  • 이미지 최적화: Next.js Image 컴포넌트 사용 (lazy loading)
  • 코드 스플리팅: Route-based splitting (자동)
  • 캐싱: API 응답 5분 캐싱 (staleTime: 5 * 60 * 1000)
