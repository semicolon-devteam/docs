# SAX v3.7.0 - 2025-11-26

### Changed

- **CHANGELOG 구조 개선** (SAX Core)
  - 단일 파일(`sax/CHANGELOG.md`) → 버전별 파일(`sax/CHANGELOG/{version}.md`) 구조로 변경
  - `sax/CHANGELOG/INDEX.md` 추가: 버전 목록 및 참조 방법 인덱스
  - 토큰 효율성 개선: 필요한 버전만 선택적으로 로드 가능
  - 총 29개 버전 파일 생성 (v1.0.0 ~ v3.6.0)

- **CLAUDE.md** (SAX-PO)
  - CHANGELOG 참조 방법 업데이트
  - 버저닝 체크리스트 개선 (INDEX.md 업데이트 추가)
  - Single Source of Truth 테이블 업데이트
  - References 섹션: CHANGELOG 링크를 INDEX.md로 변경

- **sax-architect Agent** (SAX-PO)
  - Phase 3.2: CHANGELOG 업데이트 절차 개선
  - 버저닝 체크리스트 업데이트 (CHANGELOG/{version}.md 생성, INDEX.md 업데이트)
  - 완료 보고 템플릿 업데이트

### Added

- **sax/scripts/split_changelog.py** (SAX Core)
  - CHANGELOG.md를 버전별 파일로 자동 분리하는 스크립트
  - Keep a Changelog 형식 파싱
  - 버전별 헤더 및 본문 자동 추출

### Removed

- **sax/CHANGELOG.md** (SAX Core)
  - 단일 파일 구조 제거
  - `sax/CHANGELOG/` 디렉토리 구조로 대체

### Migration Guide

**CHANGELOG 참조 방법 변경**:

1. **기존 (v3.6.0 이하)**:
   ```bash
   # 전체 CHANGELOG 읽기
   gh api repos/semicolon-devteam/docs/contents/sax/CHANGELOG.md \
     --jq '.content' | base64 -d
   ```

2. **새 방법 (v3.7.0 이상)**:
   ```bash
   # INDEX 확인 (버전 목록)
   gh api repos/semicolon-devteam/docs/contents/sax/CHANGELOG/INDEX.md \
     --jq '.content' | base64 -d

   # 특정 버전 CHANGELOG 읽기
   gh api repos/semicolon-devteam/docs/contents/sax/CHANGELOG/3.7.0.md \
     --jq '.content' | base64 -d
   ```

**변경 이유**:

- 토큰 효율성: 필요한 버전만 선택적으로 로드
- 가독성 향상: 버전별로 독립적인 파일
- 관리 용이성: 새 버전 추가 시 기존 내용 수정 불필요
- 확장성: 버전이 증가해도 단일 파일 크기 문제 없음

**주요 개선**:

- 단일 CHANGELOG.md (886줄) → 29개 버전 파일 (평균 ~30줄)
- 최신 버전 참조 시 토큰 사용량 약 95% 감소
- INDEX.md로 전체 버전 히스토리 한눈에 확인 가능

**버저닝 절차 업데이트**:

기존:
1. `sax/VERSION` 업데이트
2. `sax/CHANGELOG.md` 상단에 새 버전 추가
3. 커밋

변경:
1. `sax/VERSION` 업데이트
2. `sax/CHANGELOG/{version}.md` 생성
3. `sax/CHANGELOG/INDEX.md` 업데이트 (Latest Version, Version History)
4. 커밋

**호환성**:

- 기존 `sax/CHANGELOG.md` 참조는 404 에러 발생
- `sax/CHANGELOG/INDEX.md` 또는 버전별 파일 참조로 변경 필요
