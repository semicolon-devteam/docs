# SAX v3.14.0 - Skills Progressive Disclosure 대규모 개편

**Release Date**: 2025-11-27

## Overview

SAX-Next 및 SAX-PO 패키지의 모든 Skills에 **Anthropic Skills 표준**을 적용하여 Progressive Disclosure 패턴으로 전면 개편했습니다. 총 16개 Skills의 SKILL.md 파일을 60 lines 이하로 압축하고, 상세 내용은 references/ 디렉토리로 분리하여 컨텍스트 효율성을 대폭 개선했습니다.

### Frontmatter 표준화

모든 Skills의 Frontmatter description에 **"Use when"** 패턴 적용:

```yaml
# Before
description: Execute ADD Phase 4 with phased development

# After
description: Execute ADD Phase 4 with phased development (v0.0.x → v0.4.x). Use when (1) specification docs are complete, (2) user requests feature implementation, (3) implementing DDD 4-layer with TDD and Supabase patterns.
```

## Rationale

### 기존 문제점

- **컨텍스트 과부하**: 200-300 lines SKILL.md 파일이 Claude 컨텍스트를 과도하게 점유
- **Anthropic 표준 미준수**: Frontmatter description에 "when to use" 패턴 누락
- **Progressive Disclosure 미적용**: 상세 내용이 main 파일에 혼재

### 개선 사항

- **컨텍스트 효율성 70% 향상**: 평균 200 lines → 60 lines
- **Anthropic Skills 표준 100% 준수**: Frontmatter, Progressive Disclosure 완전 적용
- **검색 최적화**: main 파일에 핵심 정보만 노출, 상세는 references/로 분리

## Technical Details

### Progressive Disclosure 패턴

**SKILL.md (Main)** 구조:
1. Frontmatter (name, description with "Use when", tools)
2. Purpose (1줄)
3. When to Use (bullet points)
4. Quick Start / Phase Overview (테이블)
5. Critical Rules
6. Related Skills
7. References (references/ 링크)

**references/** 구조:
- workflow.md / phase-workflow.md: 상세 워크플로우
- output-format.md: 출력 형식, JSON 스키마
- domain-specific.md: 도메인별 상세 정보

### Frontmatter "Use when" 패턴

```yaml
description: |
  {핵심 기능 설명}. Use when
  (1) {첫 번째 사용 케이스},
  (2) {두 번째 사용 케이스},
  (3) {세 번째 사용 케이스}.
```

## Migration Guide

### Skill 상세 정보 참조

**Before**:
```markdown
# SKILL.md 파일 전체 읽기
```

**After**:
```markdown
# Main 정보
skill:implement

# 상세 정보 필요 시
Read: references/phase-workflow.md
Read: references/commit-strategy.md
```
