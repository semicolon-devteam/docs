# 3.25.0 (2025-11-27)

## Summary

docs 레포지토리 `.claude/` 구조 리팩토링 - SAX-Meta 전용화

## Added

### SAX 개발 필수 원칙 3번 추가

SAX-Meta CLAUDE.md에 "3. 작업 완료 후 버저닝 체크 필수 원칙" 추가:

- 모든 SAX 작업 완료 후 버저닝 필요 여부 반드시 체크
- 변경 유형별 버전 타입 명시 (MAJOR/MINOR/PATCH)
- 작업 완료 후 필수 출력 포맷 정의
- 버저닝 미실행 시 경고 안내

## Changed

### `.claude/` 구조 단순화

- **패키지 통합**: docs 레포지토리에서 SAX-Meta 패키지만 사용하도록 변경
- **CLAUDE.md 역할 변경**: 패키지 선택기 → SAX-Meta 직접 진입점
- **동기화 규칙 간소화**: sax-core, sax-meta만 동기화 대상

### 구조 변경 (Before → After)

```
Before:
.claude/
├── CLAUDE.md           # 패키지 선택기
├── sax-core/           # SAX Core 규칙
├── sax-po/             # PO/기획자용 (88 files)
├── sax-meta/           # SAX 개발자용
└── sax-next/           # Next.js 개발자용 (102 files)

After:
.claude/
├── CLAUDE.md           # SAX-Meta 직접 진입점
├── sax-core/           # SAX Core 규칙
└── sax-meta/           # SAX-Meta 패키지
```

## Rationale

### 문제점

1. **Agent/Skill 이름 충돌**: orchestrator, teacher, onboarding-master 등이 여러 패키지에 중복
2. **Claude Code 동작**: `.claude/*/agents/`, `.claude/*/skills/` 자동 스캔으로 예측 불가능한 동작
3. **불필요한 컨텍스트**: docs 레포에서 PO/Next.js 워크플로우 불필요

### 해결책

- docs 레포 = SAX-Meta 전용 (SAX 패키지 관리/개발)
- SAX-PO, SAX-Next는 각각 별도 레포지토리에 배포
- Agent/Skill 이름 충돌 완전 해소

### 효과

- 파일 수: 228개 → 38개 (84% 감소)
- Agent/Skill 충돌: 5개 → 0개
- 컨텍스트 명확성 확보

## Migration Guide

### docs 레포지토리 사용자

- `[SAX]` 또는 `[Semicolon AX]` 키워드 없이 바로 SAX-Meta 사용 가능
- 기존 SAX-Meta 워크플로우 동일하게 동작

### PO/기획자

- docs 레포에서 SAX-PO 사용 불가
- 별도 레포지토리에서 SAX-PO 배포 후 사용

### Next.js 개발자

- docs 레포에서 SAX-Next 사용 불가
- 별도 레포지토리에서 SAX-Next 배포 후 사용
