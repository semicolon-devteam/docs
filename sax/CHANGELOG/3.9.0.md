# SAX v3.9.0 - 2025-11-26

### Changed

- **.claude/ 패키지 구조 개선** (docs 레포 전용)
  - 기존: `.claude/` (단일 SAX-PO 패키지)
  - 변경: `.claude/CLAUDE.md` (패키지 선택 로직) + `.claude/sax-po/` + `.claude/sax-meta/`

- **패키지 선택 로직 분리**
  - `.claude/CLAUDE.md`: 패키지 선택 진입점
  - `Semicolon AX` 또는 `SAX` + 두 줄 띄우기 → SAX-Meta 활성화
  - 기타 모든 요청 → SAX-PO 활성화 (기본값)

- **SAX-PO orchestrator.md**
  - Routing Decision Table에서 "SAX 메타 작업" 라우팅 제거
  - 메타 작업은 루트 CLAUDE.md에서 패키지 선택으로 처리

- **동기화 규칙 업데이트**
  - SAX-PO: `sax/packages/sax-po/` → `.claude/sax-po/`
  - SAX-Meta: `sax/packages/sax-meta/` → `.claude/sax-meta/`
  - 기존 동일 위치 동기화 → 서브디렉토리 동기화로 변경

### Migration Guide

**docs 레포 사용자**:

1. **새로운 패키지 구조 이해**:
   ```
   .claude/
   ├── CLAUDE.md          # 패키지 선택 로직
   ├── sax-po/            # PO/기획자용 (기본값)
   │   ├── CLAUDE.md
   │   ├── agents/
   │   ├── skills/
   │   ├── commands/
   │   └── templates/
   └── sax-meta/          # SAX 개발자용
       ├── CLAUDE.md
       ├── agents/
       └── skills/
   ```

2. **SAX-Meta 사용 방법**:
   ```
   Semicolon AX

   새 Agent 추가해줘
   ```
   또는
   ```
   SAX

   버저닝 해줘
   ```

3. **SAX-PO 사용 방법** (기존과 동일):
   ```
   Epic 생성해줘
   ```
   별도 키워드 없이 평소처럼 사용

4. **동기화 방법 변경**:
   - 기존: `sax/packages/sax-po/` → `.claude/`
   - 변경: `sax/packages/sax-po/` → `.claude/sax-po/`
   - 추가: `sax/packages/sax-meta/` → `.claude/sax-meta/`

**주요 변경 이유**:

1. **관심사 명확한 분리**: PO 작업 ↔ SAX 메타 작업 패키지 레벨 분리
2. **명시적 선택**: 루트 CLAUDE.md에서 패키지 선택 로직 처리
3. **확장성**: 향후 SAX-Spring, SAX-Next 등 추가 패키지 병렬 구성 가능
4. **Orchestrator 단순화**: 각 패키지 orchestrator는 도메인 라우팅만 담당
