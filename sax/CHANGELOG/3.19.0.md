# Changelog v3.19.0

All notable changes to SAX packages in version 3.19.0.

## [3.19.0] - 2025-11-27

### Changed

- **SAX-Next**: Agent 통폐합 - `database-specialist` + `supabase-architect` → `database-master`
  - 역할 중복 해소: 두 Agent 모두 Supabase/DB 작업을 담당하던 것을 단일 Agent로 통합
  - Schema 설계, Migration, RLS, RPC, Repository 패턴을 하나의 Agent에서 관리

### Removed

- **database-specialist.md**: database-master로 통합됨
- **supabase-architect.md**: database-master로 통합됨

### Added

- **database-master.md**: 통합된 Database 및 Supabase 전문 Agent
  - Schema 설계 및 Migration 생성
  - RLS 정책 구현
  - RPC 함수 활용 및 Repository 패턴 구현
  - core-supabase 패턴 준수 검증
  - Type-safe 쿼리 구현

## Summary

SAX-Next 패키지의 중복 Agent를 통폐합하여 역할 명확화 및 라우팅 효율성을 개선했습니다.

**통폐합 내역**:
| Before | After |
|--------|-------|
| database-specialist (45 lines) | database-master (160 lines) |
| supabase-architect (238 lines) | (통합됨) |

**효과**:
- Orchestrator 라우팅 단순화 (2개 → 1개)
- 역할 중복 해소
- 사용자 혼란 감소

## Migration Guide

기존에 `database-specialist` 또는 `supabase-architect`를 직접 호출하던 경우:

```diff
- [SAX] Agent: database-specialist 호출
- [SAX] Agent: supabase-architect 호출
+ [SAX] Agent: database-master 호출
```
