### Migration Guide

**SAX-PO/SAX-Next 사용자**:

기존 사용자에게는 영향 없음. SAX 메타 작업 관련 컴포넌트가 별도 패키지로 분리되었으나, 일반 사용자는 SAX-Meta를 직접 사용할 필요 없음.

**SAX 개발자 (프레임워크 관리자)**:

1. SAX 패키지 구조 변경 시 SAX-Meta 참조:
   ```bash
   # SAX-Meta CLAUDE.md 참조
   gh api repos/semicolon-devteam/docs/contents/sax/packages/sax-meta/CLAUDE.md \
     --jq '.content' | base64 -d
   ```

2. Agent/Skill 생성 시 agent-creator, skill-creator 사용

3. 버저닝 시 version-manager Skill 활용

4. 패키지 검증 시 package-validator Skill 사용

**주요 변경 이유**:

- **관심사 분리**: PO/개발자 도구 ↔ 프레임워크 관리 도구 명확히 분리
- **중복 제거**: SAX-PO와 SAX-Next에 각각 존재하던 skill-creator 통합
- **확장성**: SAX 패키지 관리 도구를 한곳에 모아 유지보수 용이

## Package Structure

```
sax/packages/
├── sax-po/          # PO/기획자용 (Epic, Draft Task, Spec)
├── sax-next/        # Next.js 개발자용 (SDD, ADD, DDD)
└── sax-meta/        # SAX 개발자용 (패키지 관리, 신규)
    ├── agents/
    │   ├── sax-architect.md      # 이동
    │   ├── command-creator.md    # 이동
    │   └── agent-creator.md      # 신규
    └── skills/
        ├── skill-creator/        # 이동
        ├── package-validator/    # 신규
        └── version-manager/      # 신규
```
