# SAX v3.21.0 - 2025-11-27

## Summary

SAX 패키지 배포 자동화 Skill 추가. `package-sync`와 `package-deploy` Skill을 통해 패키지 동기화 및 외부 프로젝트 배포를 자동화합니다.

## Added

### SAX-Meta Skills

- **package-sync** (SAX-Meta)
  - SAX 패키지 소스(`sax/packages/`) → `.claude/` 디렉토리 동기화 자동화
  - docs 레포지토리 전용 Skill
  - Progressive Disclosure 적용 (SKILL.md + references/)
  - 트리거: "동기화", ".claude 동기화", "sync"

- **package-deploy** (SAX-Meta)
  - SAX 패키지를 외부 프로젝트에 배포
  - 그린필드/브라운필드 프로젝트 지원
  - deploy.sh 스크립트 래핑 및 가이드 제공
  - 트리거: "배포", "deploy", "설치", "install"

### References 문서

- `skills/package-sync/references/`
  - sync-workflow.md - 동기화 프로세스 상세
  - package-mapping.md - 패키지별 경로 매핑
  - output-format.md - 출력 형식

- `skills/package-deploy/references/`
  - deploy-workflow.md - 배포 프로세스 상세
  - target-setup.md - 대상 프로젝트 설정 가이드
  - output-format.md - 출력 형식

## Changed

- Skills 테이블에 `package-sync`, `package-deploy` 추가
- 주요 라우팅 테이블에 패키지 동기화/배포 추가

### SAX-Meta Orchestrator

- Routing Decision Table에 새 Skill 라우팅 추가:
  - `package-sync`: "동기화", ".claude 동기화", "sync"

- 워크플로우 가이드에 6, 7번 항목 추가

## Migration Guide

### 신규 Skill 사용 방법

```bash
# docs 내부 동기화 (SAX-PO, SAX-Meta)
[SAX] 패키지 동기화해줘

# 외부 프로젝트 배포
[SAX] sax-next 배포해줘 /path/to/project
```

### 기존 워크플로우와 호환

기존 `rsync` 및 `deploy.sh` 명령은 그대로 사용 가능합니다. 새 Skill은 이를 래핑하여 자동화합니다.
